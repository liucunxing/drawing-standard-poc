# 图纸信息识别及内容审查功能需求整理

## 1. 项目目标

本项目用于检查工程图纸中引用标准的适用性和准确性，降低因标准不存在、版本过期或引用错误带来的工程技术和合规风险。

用户上传 PDF 图纸后，系统自动识别图纸中的表格区域，解析表格内容，提取标准编号，并与企业标准库进行比对，输出审查结果和待人工复核项。

## 2. 业务闭环

```text
PDF 图纸上传
-> 图纸页面化
-> 表格区域识别
-> 表格区块裁剪
-> 表格结构解析
-> 标准编号提取
-> 标准号归一化
-> 企业标准库比对
-> 审查结果输出
-> 人工复核与修正
```

## 3. 核心功能

- 支持单个或多个 PDF 图纸上传。
- 图纸输入格式固定为 PDF，生产环境图纸均为超高分辨率 PDF。
- 自动识别图纸中的表格区块。
- 将表格区块解析为 Markdown/JSON 结构化结果。
- 从表格文本中提取标准编号。
- 将提取结果与企业标准库进行比对。
- 输出完全符合、年份不一致、相似匹配、不存在、待人工复核等结果。
- 支持查询主任务结果和分步骤处理结果。

## 4. 非功能要求

- 初期建议支持约 10 个并发用户。
- 上传文件较多时，应通过任务队列和进度查询避免长时间同步等待。
- 目标效果：图纸信息识别完备和准确率 95% 以上，判断识别准确率 95% 以上。

## 5. POC 边界

POC 阶段重点验证业务闭环和技术可行性，不承诺覆盖所有图纸格式，也不作为生产级准确率承诺。

图纸格式边界：

- 输入文件固定为 PDF。
- 生产图纸为超高分辨率 PDF，处理方案需要重点关注渲染 DPI、内存占用、页面切片、裁剪精度、队列任务耗时和中间文件存储。
- 不需要在 POC 阶段优先兼容 Word、图片压缩包、CAD 原文件等其他格式。

优先验证：

- 表格区域是否能被稳定找到。
- 表格裁剪是否完整。
- 表格内容是否能结构化还原。
- 标准编号是否能正确提取。
- 标准库比对结论是否可解释。
