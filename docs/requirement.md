# 图纸信息识别及内容审查功能需求整理

## 1. 项目目标

本项目用于检查工程图纸中引用标准的适用性和准确性，降低因标准不存在、版本过期或引用错误带来的工程技术和合规风险。

用户上传 PDF 图纸后，系统自动识别图纸中的表格区域和图纸元数据区域，解析表格内容，提取标准编号，并与企业标准库进行比对，输出审查结果和待人工复核项。图纸基本信息应同步沉淀到数据库，支持后续检索、统计和元数据管理。

## 2. 业务闭环

```text
PDF 图纸上传
-> 图纸页面化
-> 图纸元数据识别
-> 表格区域识别
-> 表格区块裁剪
-> 表格结构解析
-> 标准编号提取
-> 标准号归一化
-> 企业标准库比对
-> 审查结果输出
-> 人工复核与修正
```

## 3. 核心功能

- 支持单个或多个 PDF 图纸上传。
- 图纸输入格式固定为 PDF，生产环境图纸均为超高分辨率 PDF。
- 自动识别图纸中的图纸元数据，例如项目名称、装置/设备名称、图号、设计阶段、专业、比例、页码、日期、签审信息等。
- 自动识别图纸中的全部候选表格区块，并保留截图、bbox、页码、置信度和分类结果。
- 将表格区块解析为 Markdown/JSON 结构化结果。
- 从全部表格文本和关键文本区域中提取标准编号。
- 将提取结果与企业标准库进行比对。
- 输出完全符合、年份不一致、相似匹配、不存在、待人工复核等结果。
- 支持查询主任务结果和分步骤处理结果。

## 4. 表格识别范围

工程图纸版式不固定，表格位置和形态可能随图纸类型变化。POC 不应只裁剪固定位置或单一目标表格，而应优先识别并保存全部候选表格，再通过后续规则、标准号抽取和业务字段识别筛选有效内容。

候选表格建议分为：

| 类型 | 说明 | 处理策略 |
| --- | --- | --- |
| 标准/技术参数表 | 包含制造、检验、设计数据、壳程/管程等信息 | 必须解析并参与标准审查 |
| 管口表 | 包含管口编号、法兰标准、用途、尺寸等信息 | 必须解析，可能包含标准号 |
| 材料/明细表 | 包含材料牌号、材料标准、件号、数量等信息 | 必须解析，可能包含标准号 |
| 标题栏/签审栏 | 包含图号、项目、设备名称、签审日期等元数据 | 作为图纸元数据抽取，不作为标准审查主表 |
| 技术要求文本块 | 包含条款式说明和引用标准 | 应进行文本 OCR/标准号抽取，不强制按表格解析 |
| 其它候选表格 | 局部详图说明框、空白表格、辅助信息框 | 保留中间结果，低优先级解析或人工复核 |

## 5. 非功能要求

- 初期建议支持约 10 个并发用户。
- 上传文件较多时，应通过任务队列和进度查询避免长时间同步等待。
- 目标效果：图纸信息识别完备和准确率 95% 以上，判断识别准确率 95% 以上。

## 6. POC 边界

POC 阶段重点验证业务闭环和技术可行性，不承诺覆盖所有图纸格式，也不作为生产级准确率承诺。

图纸格式边界：

- 输入文件固定为 PDF。
- 生产图纸为超高分辨率 PDF，处理方案需要重点关注渲染 DPI、内存占用、页面切片、裁剪精度、队列任务耗时和中间文件存储。
- 不需要在 POC 阶段优先兼容 Word、图片压缩包、CAD 原文件等其他格式。

优先验证：

- 表格区域是否能被稳定找到。
- 标题栏/签审栏中的图纸元数据是否能被稳定抽取。
- 表格裁剪是否完整。
- 表格内容是否能结构化还原。
- 标准编号是否能正确提取。
- 标准库比对结论是否可解释。
