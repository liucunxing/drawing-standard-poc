# 处理流程设计

## 1. 主流程

```text
用户上传 PDF
-> 创建审查任务
-> PDF 按指定 DPI 转页面图像
-> 页面版式识别
-> 表格区域检测
-> 表格图片裁剪
-> 表格结构解析
-> 生成 Markdown/JSON
-> 标准编号提取
-> 标准编号归一化
-> 标准库比对
-> LLM 辅助解释/纠错
-> 结果汇总
-> 用户查询与人工复核
```

## 2. 任务状态

建议任务状态如下：

| 状态 | 说明 |
| --- | --- |
| pending | 已创建，等待处理 |
| processing | 正在处理 |
| success | 处理完成 |
| failed | 处理失败 |
| review_required | 存在待人工复核项 |

## 3. 超高分辨率 PDF 处理注意事项

生产输入固定为 PDF，且图纸分辨率极高。处理流程不要简单按普通文档 PDF 处理。

建议策略：

- PDF 渲染 DPI 需要配置化，POC 阶段建议测试多档 DPI 的识别效果和耗时。
- 页面图像可能很大，应避免一次性把多页大图全部加载进内存。
- 表格检测可先使用适当降采样页面图，定位 bbox 后再回到高分辨率图进行精确裁剪。
- 所有 bbox 坐标需要记录对应的渲染 DPI、缩放比例和页面尺寸，避免后续裁剪错位。
- 中间文件需要分任务、分页、分表格保存，便于失败重试和人工排查。
- 上传、识别、解析、比对建议走异步任务队列，不建议长时间阻塞 HTTP 请求。
- 需要设置单文件大小、单任务页数、最大处理时长和失败重试策略。

## 4. 分步骤结果

每个任务应保留以下中间结果，方便 PM/BA、开发和业务专家定位问题：

- 原始 PDF 文件路径。
- 页面图像路径。
- 表格区块 bbox、页码、截图路径。
- 表格解析 Markdown/JSON。
- 抽取出的标准编号原文。
- 标准号归一化结果。
- 标准库匹配候选。
- 最终审查结论。

## 5. 推荐优先级

POC 阶段先做离线闭环，再做页面：

1. 固定样本 PDF 输入。
2. 输出表格截图。
3. 输出表格 Markdown。
4. 输出标准清单。
5. 输出标准库比对结果。
6. 补充 Web 上传、任务查询和结果展示。
