# 处理流程设计

## 1. 主流程

```text
用户上传 PDF
-> 创建审查任务
-> PDF 转页面图像
-> 页面版式识别
-> 表格区域检测
-> 表格图片裁剪
-> 表格结构解析
-> 生成 Markdown/JSON
-> 标准编号提取
-> 标准编号归一化
-> 标准库比对
-> LLM 辅助解释/纠错
-> 结果汇总
-> 用户查询与人工复核
```

## 2. 任务状态

建议任务状态如下：

| 状态 | 说明 |
| --- | --- |
| pending | 已创建，等待处理 |
| processing | 正在处理 |
| success | 处理完成 |
| failed | 处理失败 |
| review_required | 存在待人工复核项 |

## 3. 分步骤结果

每个任务应保留以下中间结果，方便 PM/BA、开发和业务专家定位问题：

- 原始 PDF 文件路径。
- 页面图像路径。
- 表格区块 bbox、页码、截图路径。
- 表格解析 Markdown/JSON。
- 抽取出的标准编号原文。
- 标准号归一化结果。
- 标准库匹配候选。
- 最终审查结论。

## 4. 推荐优先级

POC 阶段先做离线闭环，再做页面：

1. 固定样本 PDF 输入。
2. 输出表格截图。
3. 输出表格 Markdown。
4. 输出标准清单。
5. 输出标准库比对结果。
6. 补充 Web 上传、任务查询和结果展示。
