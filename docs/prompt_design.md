# LLM Prompt 设计草案

## 1. 使用原则

LLM 是辅助模块，不作为唯一审查依据。标准是否存在、是否现行，应优先依据企业标准库和规则比对。

## 2. 标准号提取 Prompt 目标

输入 Markdown 表格文本，输出标准编号清单。

要求：

- 只提取疑似标准编号。
- 保留原始文本。
- 给出归一化建议。
- 不要编造表格中不存在的标准。

建议输出 JSON：

```json
[
  {
    "original_text": "NB/T 47041-2014",
    "normalized_code": "NB/T 47041",
    "standard_year": "2014",
    "confidence": 0.95
  }
]
```

## 3. 相似候选解释 Prompt 目标

输入：

- 图纸识别出的标准号。
- 标准库候选列表。
- 规则比对结果。

输出：

- 是否可能为 OCR 错误。
- 最可能候选。
- 面向用户的解释。
- 是否建议人工复核。

## 4. 审查意见生成 Prompt 目标

输入标准比对结果，生成简洁可读的审查意见。

示例：

```text
图纸引用 NB/T 47041-2014，标准库现行版本为 NB/T 47041-2022，建议复核是否应更新引用版本。
```
