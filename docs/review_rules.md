# 标准比对判定规则

## 1. 判定结果类型

| 结果类型 | 判定口径 |
| --- | --- |
| 完全符合 | 标准主编号和年份均与标准库现行记录一致 |
| 年份不一致 | 标准主编号存在，但图纸引用年份与标准库现行年份不一致 |
| 相似匹配 | 标准库中存在相似编号，可能存在 OCR 误识别或书写差异 |
| 不存在 | 标准库中找不到该标准编号或可接受候选 |
| 待人工复核 | 置信度低、候选冲突或规则无法稳定判断 |

## 2. 标准号归一化原则

归一化时应尽量拆分为：

- 标准体系/前缀，例如 GB/T、NB/T、HG/T、SH/T。
- 主编号，例如 150、47041、20592。
- 分册号，例如 150.1、47018.4。
- 年份，例如 2011、2022。

示例：

| 原始文本 | 归一化编号 | 年份 |
| --- | --- | --- |
| GB/T 150.1 | GB/T 150.1 |  |
| NB/T 47041-2014 | NB/T 47041 | 2014 |
| NB/T 47018.4-2022 | NB/T 47018.4 | 2022 |

## 3. LLM 使用边界

LLM 只作为辅助判断，不直接替代标准库比对结果。

适合使用 LLM 的场景：

- OCR 结果疑似混淆时，辅助纠错。
- 多个相似候选时，解释差异。
- 生成面向用户的审查意见。

不建议使用 LLM 的场景：

- 直接判断某标准是否现行。
- 直接覆盖数据库精确匹配结论。
- 在没有原始识别文本和标准库候选的情况下单独生成结论。
