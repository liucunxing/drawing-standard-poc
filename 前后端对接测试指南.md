# 前后端对接测试指南

## 功能说明
已完成PDF上传功能的前后端对接:
- **前端**: Streamlit应用 (streamlit_app.py)
- **后端**: FastAPI应用 (drawing-standard-poc/backend)
- **数据库**: MySQL (通过SQLManager操作)
- **功能**: 用户通过前端上传PDF文件,后端接收并保存到 `tmp/uploads` 目录,同时记录到数据库


## 代码架构

### 调用链路:

```
前端 (streamlit_app.py)
  ↓ upload_pdf_to_backend()
  ↓ HTTP POST /api/drawing/upload-pdf
后端 routes.py
  ↓ poc_service.upload_pdf()
poc_service.py
  ↓ TableLayoutService4Batch.base_dir
  ↓ 保存文件到 tmp/uploads/
  ↓ SQLManager.create() 
  ↓ 插入记录到 pdf_task 表
返回结果给前端 (包含task_id和db_record_id)
```

### 关键文件:

1. **poc_service.py** - 业务逻辑层
   - `upload_pdf()`: 接收PDF,生成task_id,保存文件,记录数据库
   - `get_task_status()`: 查询任务状态
   - `list_tasks()`: 查询任务列表
   - `_save_to_database()`: 使用SQLManager插入pdf_task记录

2. **routes.py** - API路由层
   - `POST /api/drawing/upload-pdf`: 接收文件,调用poc_service

3. **streamlit_app.py** - 前端展示层
   - `upload_pdf_to_backend()`: 调用后端API
   - `render_upload()`: 上传界面,显示结果

## 常见问题

### Q1: 前端显示"无法连接到后端服务器"
**解决**: 
- 确认后端已启动: 访问 `http://localhost:8000/docs`
- 检查端口是否被占用
- 确认 `BACKEND_BASE_URL` 配置正确

### Q2: 上传失败,提示"仅支持PDF文件"
**解决**: 
- 确认上传的是 `.pdf` 格式文件
- 检查文件扩展名是否正确

### Q3: 文件保存到哪里了?
**位置**: 
```
D:\work\Develop\drawing-poc\drawing-standard-poc\backend\tmp\uploads\
```

### Q4: task_id是如何生成的?
**规则**: `{文件名前缀}_{时间戳}`
**示例**: `02.194-1-A1(E-420310)_20260529143025`
